## ADDED Requirements

### Requirement: Source landing page remains visually balanced on mobile
The system SHALL render the source landing page with a mobile-optimized header and hero layout so that source-acquisition content remains readable and visually balanced on small screens.

#### Scenario: Source page header navigation stays legible on mobile
- **WHEN** the source landing page is viewed on a narrow mobile screen
- **THEN** the brand block and navigation are stacked or spaced to prevent cramped wrapping
- **AND** each navigation item remains clearly distinguishable and tappable

#### Scenario: Source page hero content stacks cleanly on mobile
- **WHEN** the source landing page hero is viewed on a narrow mobile screen
- **THEN** the service banner, heading, description, and supporting panel stack with deliberate spacing
- **AND** the page does not display squeezed text blocks or uneven CTA compression in the hero area

### Requirement: Source download actions stay clear on mobile
The system SHALL present each source download card with mobile-friendly action controls and readable download metadata.

#### Scenario: Download card actions avoid horizontal compression
- **WHEN** a source download card is viewed on a narrow mobile screen
- **THEN** its buttons and metadata stack or wrap in a controlled layout
- **AND** the card keeps both acquisition methods readable without overflow or crowded alignment
